# SKILL: Create a SKILL.md from a Conversation

## Summary

This skill extracts a repeatable workflow from a conversation and produces a concise, usable `SKILL.md` file for agent customization. It is intended to be used in a workspace (repository-scoped) to capture the team's conversational workflow, decision points, and completion criteria.

## When to run

- After a multi-step pair-programming session, design discussion, or debugging conversation.
- When a recurring workflow, checklist, or runbook emerges from chat and should be codified.

## Inputs

- Full conversation history (chat transcript) or a description of the steps already taken.

## Outputs

- A draft `SKILL.md` saved to the workspace root.
- A short list of clarifying questions (if any) needed to finalize the skill.

## Extraction Procedure (step-by-step)

1. Review conversation context end-to-end.
2. Identify explicit steps performed or recommended (list them in order).
3. Identify decision points and branching logic (conditions -> outcomes).
4. Capture quality criteria and completion checks (how to know task is done).
5. Note any required inputs, files, or environmental assumptions.

## Clarifying Questions (templates)

- What final outcome should this skill produce (artifact, file, or state)?
- Should the skill be workspace-scoped or user-personal?
- Prefer a short checklist or a full multi-step workflow with examples?

## Drafting & Iteration

1. Draft the skill using the extraction results (include steps, decision points, and checks).
2. Save the draft as `SKILL.md` in the repo root.
3. Highlight ambiguous or missing parts as explicit clarifying questions.
4. Ask the user the clarifying questions, incorporate answers, and update the draft.
5. Finalize and mark the skill ready for use.

## Acceptance Criteria / Quality Checklist

- Step sequence is complete and ordered.
- Decision branches are explicit and actionable.
- Completion checks are measurable (tests, files, states).
- Examples or sample prompts are included for quick testing.
- The file is saved in the workspace and linked to usage instructions.
